SoftAdmin

SoftAdmin es una solución integral desarrollada en colaboración con un equipo especializado en el núcleo de negocio de Bersamed, una aplicación de escritorio. Su objetivo principal es facilitar la comunicación en tiempo real entre los distintos equipos de trabajo y ofrecer un seguimiento visual del estado de cada tarea o proyecto.
Tanto entre los equipos de trabajo y los clientes.

Descripción General del Proyecto

La problemática

Bersamed requería una solución integral que permitiera coordinar sus procesos internos en tiempo real entre distintos equipos de trabajo. El sistema anterior no lograba centralizar la información ni brindar comunicación instantánea, lo que ocasionaba retrasos y errores en el flujo de trabajo. Los principales retos fueron:

  • Falta de comunicación en tiempo real entre áreas clave, provocando descoordinación.
  • Ausencia de una plataforma unificada para seguimiento de trabajos y estado de tareas.
  • Procesos manuales que incrementaban el riesgo de errores y pérdida de información.
  • Limitaciones técnicas para escalar el sistema y adaptarlo a nuevas necesidades.

La solución

SoftAdmin es una plataforma integrada que permite el seguimiento y gestión en tiempo real de trabajos técnicos, asegurando sincronización inmediata y soporte offline para mantener la productividad aún sin conexión.

  • Implementación de un backend en Spring Boot que expone API REST seguras con JWT y roles para controlar accesos.
  • Comunicación en tiempo real mediante Server-Sent Events (SSE) para notificaciones instantáneas de cambios de estado.
  • Integración con Firebase Cloud Messaging (FCM) para enviar notificaciones push cuando la app Android está en segundo plano o cerrada.
  • Aplicación Android con persistencia local usando SQLite y sincronización diferida para funcionamiento offline fiable.
  • Panel web desarrollado en Angular para gestión completa de trabajos, asignaciones y visualización en tiempo real.

Resultado e impacto

SoftAdmin logró optimizar significativamente la gestión y seguimiento de trabajos técnicos con resultados visibles en diferentes áreas:

Actualización en tiempo real
Notificaciones instantáneas que mejoran la comunicación entre técnicos y supervisores.
Funcionalidad offline robusta
Permite trabajar sin conexión y sincronizar datos cuando hay disponibilidad.
Seguridad y control de acceso
Accesos protegidos mediante JWT y roles que garantizan integridad de la información.
Mayor productividad
Reducción de errores y tiempos muertos en asignaciones y reportes de trabajo.

Implementación Técnica

Arquitectura Backend(Servidor)

Controlador y Servicios REST

Implementación de controladores REST en Spring Boot para gestionar operaciones CRUD de trabajos, asignaciones y usuarios. Servicios desacoplados para lógica de negocio, utilizando inyección de dependencias para mantener código modular y testeable.

Módulo de Tiempo Real (SSE y FCM)

Implementación de Server-Sent Events (SSE) para emitir actualizaciones en tiempo real a clientes web y Android, filtradas por roles y permisos. Integración con Firebase Cloud Messaging (FCM) para notificaciones push cuando la app está en segundo plano o cerrada.

Integración con Base de Datos MySQL

Uso de Spring Data JPA para interacción con MySQL, incluyendo manejo eficiente de relaciones entre entidades (trabajos, usuarios, asignaciones). Estrategias de optimización y migraciones controladas.

Seguridad y Autenticación

Configuración de Spring Security con JWT para autenticación y autorización basada en roles (admin, técnico). Control estricto de acceso a recursos y refresco de tokens para mantener sesiones seguras.

Arquitectura Android

Capa de UI y Estado

Interfaces construidas con Jetpack Compose para UI declarativa y reactiva. Gestión de estado usando ViewModel y LiveData para mantener datos persistentes y actualizados.

Comunicación con Backend

Retrofit para llamadas HTTP REST seguras con autenticación JWT. Consumo de Server-Sent Events (SSE) para actualizaciones en tiempo real. Integración con Firebase Cloud Messaging (FCM) para notificaciones push.

Persistencia Local

SQLite Database para almacenamiento local estructurado con soporte para sincronización diferida. WorkManager gestiona tareas en segundo plano, reintentos y sincronización segura para garantizar datos consistentes.

Gestión de Seguridad y Sesiones

Manejo de tokens JWT para autenticación y autorización con almacenamiento seguro. Renovación automática de tokens y control de acceso basado en roles para proteger recursos sensibles.

Web App para Clientes y Empleados

Frontend Layer

Aplicación web desarrollada en Angular con gestión de estado reactiva y diseño responsivo usando Material Design. Interfaz adaptada para roles de clientes y empleados, mostrando solo la información permitida.

API Layer

APIs RESTful desarrolladas en Spring Boot con autenticación JWT y autorización basada en roles. Control granular que permite a los clientes acceder solo a sus propios servicios y actualizaciones, mientras que los empleados tienen acceso a toda la información necesaria para gestión.

Base de Datos

MySQL gestionada vía Spring Data JPA con esquemas optimizados para separar y filtrar datos por usuario y rol. Estrategias para mantener integridad, seguridad y rendimiento en consultas multi-rol.

@RestController
@RequestMapping("/login")
public class ControladorAuth 
{
    private final AutenticacionServicio autenticacionServicio;
    private final RateLimiting rateLimiting = new RateLimiting();

    public ControladorAuth(AutenticacionServicio autenticacionServicio) {
        this.autenticacionServicio = autenticacionServicio;
    }

    @PostMapping("/trabajadores")
    public ResponseEntit<String> loginTrabajadores(@RequestBody PeticionLoginTrabajadores peticion, HttpServletRequest request) throws Exception 
    {
        String ip = request.getRemoteAddr();

        //valida la cantidad de peticiones ya hechas
        if (!rateLimiting.permitirPeticion(ip)) {
            return ResponseEntity.status(429).body("Demasiados intentos. Intenta de nuevo en un momento.");
        }

        //Envía el JWT
        try {
            String token = autenticacionServicio.iniciarSesionTrabajador(peticion.getUsuario(), peticion.getPassword());
            return ResponseEntity.ok(token);
        } 
        catch (RuntimeException ex) {
            return ResponseEntity.status(401).body(ex.getMessage());
        } catch (Exception e) {
            return ResponseEntity.status(500).body("Error interno del servidor");
        }
    }
    
    
    @PostMapping("/clientes")
    public ResponseEntity<String> loginClientes(@RequestBody PeticionLoginClientes peticion, HttpServletRequest request) throws Exception 
    {
        String ip = request.getRemoteAddr();


        //valida la cantidad de peticiones ya hechas
        if (!rateLimiting.permitirPeticion(ip)) {
            return ResponseEntity.status(429).body("Demasiados intentos. Intenta de nuevo en un momento.");
        }
       
        //Envía el JWT
        try {
            String token = autenticacionServicio.iniciarSesionCliente(peticion.getRazonSocial(), peticion.getRfc());
            return ResponseEntity.ok(token);
        } catch (RuntimeException ex) {
            return ResponseEntity.status(401).body(ex.getMessage());
        } catch (Exception e) {
            return ResponseEntity.status(500).body("Error interno del servidor");
        }
    }
}

private void autenticarUsuario(String usuario, String contraseña)
{
    runOnUiThread(() ->
    {
        InputMethodManager imm = (InputMethodManager) getSystemService(Activity.INPUT_METHOD_SERVICE);
        View view = getCurrentFocus();
        if (view == null) {
            view = new View(this);
        }
        imm.hideSoftInputFromWindow(view.getWindowToken(), 0);

        ProgressBar loading = findViewById(R.id.loading);
        loading.setVisibility(View.VISIBLE);
    });

    new Thread(() -> 
    {
        try {
            URL url = new URL(getString(R.string.autenticationURL));
            HttpURLConnection conn = (HttpURLConnection) url.openConnection();
            conn.setRequestMethod("POST");
            conn.setConnectTimeout(3000);
            conn.setReadTimeout(3000);
            conn.setRequestProperty("Content-Type", "application/json");
            conn.setDoOutput(true);

            JSONObject credenciales = new JSONObject();
            credenciales.put("usuario", usuario);
            credenciales.put("password", contraseña);

            try (OutputStream os = conn.getOutputStream()) {
                os.write(credenciales.toString().getBytes(StandardCharsets.UTF_8));
            }

            int responseCode = conn.getResponseCode();

            InputStream is = (responseCode == 200)
                    ? conn.getInputStream()
                    : conn.getErrorStream();

            BufferedReader reader = new BufferedReader(new InputStreamReader(is));
            StringBuilder response = new StringBuilder();
            String line;

            while ((line = reader.readLine()) != null) {
                response.append(line);
            }

            reader.close();
            String respuesta = response.toString();

            runOnUiThread(() -> {
                ProgressBar loading = findViewById(R.id.loading);
                loading.setVisibility(View.GONE);

                //Se logro logear
                if (responseCode == 200) {
                    SharedPreferences prefs = getSharedPreferences("auth", MODE_PRIVATE);
                    prefs.edit().putString("jwt", respuesta).apply();
                    irALaSiguienteActivity();
                } else {
                    //Error al logear
                    String mensaje = respuesta.isEmpty() ? "Error desconocido" : respuesta;
                    DialogodeAlerta(responseCode, mensaje, this);
                }
            });

        } catch (Exception e)
        {
            //Errores externos
            runOnUiThread(() -> {
                ProgressBar loading = findViewById(R.id.loading);
                loading.setVisibility(View.GONE);

                if (!e.getMessage().contains("after 3000ms")) {

                    DialogodeAlerta(520, "Ha sucedido un error inesperado, vuelva a intentar.", this);
                } else {
                    DialogodeAlerta(504, "Tiempo de espera excedido, vuelva a intentar.", this);
                }
            });
        }
    }).start();            
}

import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';
import { tap } from 'rxjs/operators';
import { HttpResponse } from '@angular/common/http';
import { throwError } from 'rxjs';
import { catchError } from 'rxjs/operators';

@Injectable({
  providedIn: 'root'
})
export class AuthService {
  private apiUrl = 'http://localhost:8080/login/clientes'; 

  constructor(private http: HttpClient) {}

login(RazonSocial: string, RFC: string): Observable<HttpResponse<string>> {
  const credentials = { RazonSocial, RFC };
  console.log('Enviando credenciales:', credentials);
  return this.http.post(this.apiUrl, credentials, {
    observe: 'response',
    responseType: 'text'
  }).pipe(
    catchError(err => {
      // Pasar el error para que lo maneje el componente
      return throwError(() => err);
    })
  );
}


  private saveToken(token: string): void {
    localStorage.setItem('jwt', token);
  }

  public getToken(): string | null {
    return localStorage.getItem('jwt');
  }

  public logout(): void {
    localStorage.removeItem('jwt');
  }
}
