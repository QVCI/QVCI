Mi Movilidad

En la categoría de Proyecto de Transporte y Movilidad Urbana, el equipo conformado por Luis Aguilar Torres, Samuel Gutiérrez López, Daniela Montaño Sánchez, Natalia Neshmi Nolasco Quezada, Abdiel Reyes Vera y Carlo Iván Quezada Venegas, obtuvo un reconocimiento de 20 mil pesos por el proyecto titulado "Sistema de Optimización de Rutas de Movilidad Integrada en la Ciudad de México". Este trabajo plantea el cálculo de rutas de movilidad compartida o integrada, con y sin conexión a internet, utilizando tecnologías como Firebase de Google para ofrecer información en tiempo real sobre líneas, mapas, estaciones, horarios, costos y seguimiento de personas que usan diferentes sistemas de transporte público. El reconocimiento fue otorgado en el marco del foro “Transporte y Ciudad: Política Nacional de Transporte Público Colectivo Urbano”, evidenciando la innovación y relevancia social del proyecto.

Descripción General del Proyecto

La problemática

Este proyecto plantea el cálculo de rutas para movilidad compartida e integrada, tanto con conexión a internet como sin ella, utilizando tecnologías como Firebase de Google para ofrecer información en tiempo real sobre líneas, mapas, estaciones, horarios, costos y seguimiento de usuarios en diferentes sistemas de transporte público. A pesar de que existen sistemas que consideran datos en tiempo real y mapas, el valor principal de esta solución es la integración y centralización de la información, ya que actualmente la información oficial está dispersa, desactualizada o presenta inconsistencias entre distintas fuentes gubernamentales.

  • Dificultad para compartir ubicación en tiempo real, lo que impacta en la seguridad de los usuarios.
  • Falta de opciones confiables para cálculo y visualización de rutas offline.
  • Fragmentación y desactualización de la información oficial sobre transporte público en diferentes plataformas.
  • Necesidad de centralizar datos y mejorar la experiencia del usuario con una plataforma unificada.

La solución

La solución desarrollada es una plataforma integrada que permite el cálculo y visualización de rutas para movilidad compartida e integrada, con soporte para uso offline y en tiempo real. Utiliza Firebase para ofrecer información actualizada sobre líneas, estaciones, horarios, costos y el seguimiento en vivo de usuarios para mejorar su seguridad y experiencia.

  • Cálculo de rutas óptimas integrando distintas modalidades de transporte, con y sin conexión a internet.
  • Visualización en tiempo real y compartición segura de la ubicación de los usuarios para mayor seguridad.
  • Centralización de información dispersa y actualización automática usando Firebase y Google Maps.
  • Soporte offline con sincronización diferida para garantizar disponibilidad constante incluso sin conexión.
  • Interfaz intuitiva para visualizar rutas y mapas que mejora la experiencia del usuario final.

Resultado e impacto

El proyecto Mi Movilidad logró optimizar significativamente la movilidad urbana en la Ciudad de México mediante una solución innovadora con funcionalidades offline y en tiempo real. Los resultados obtenidos demuestran avances en la experiencia del usuario, eficiencia en el cálculo de rutas y seguridad en el seguimiento de personas. Además, el proyecto fue reconocido con el 15° Premio Nacional de Transporte Urbano y Movilidad 2024 en la categoría de Proyecto de Transporte y Movilidad Urbana, destacando su innovación y relevancia social.

Implementación Técnica

Arquitectura Backend(Servidor)

Controlador y Servicios REST

Implementación de controladores REST en Spring Boot para gestionar los reportes de problematicas en las diversas estaciones.

Sistema de anuncios inteligentes

Procesamiento de reportes de incidentes, para evitar reportes maliciosos, se requiere un alto volumen de reportes.

Integración con Base de Datos MySQL

Uso de Spring Data JPA para interacción con MySQL, incluyendo manejo eficiente de relaciones entre entidades (ubicaciones, reportes, usuarios).

Arquitectura Android

Capa de UI y Estado

Interfaces desarrolladas con Jetpack Compose para una UI declarativa y reactiva. Gestión del estado mediante ViewModel y LiveData que garantizan persistencia y actualización eficiente.

Autenticación y Servicios en la Nube

Uso de OAuth 2.0 con Firebase Authentication para login seguro y manejo de sesiones. Firebase Realtime Database y Firestore para la compartición segura y en tiempo real de la ubicación de usuarios.

Persistencia Local y Sincronización

Almacenamiento de rutas y datos críticos en SQLite para consulta offline. WorkManager gestiona sincronización diferida y tareas en segundo plano para mantener la consistencia.

Cálculo y Visualización de Rutas

Implementación del algoritmo de Dijkstra para cálculo óptimo de rutas integradas. Visualización de mapas y rutas usando la API de Google Maps para mejorar la experiencia del usuario.

Comunicación con Backend para Incidentes

Uso de Retrofit para llamadas HTTP REST enfocadas en obtener anuncios de incidentes y alertas que afectan el transporte público.

Web App para Clientes y Empleados

Frontend Informativo y Funcional

Aplicación web desarrollada para ofrecer información relevante, con integración de pago mediante PayPal para cuentas premium. Además, permite a los usuarios descargar la aplicación móvil. Diseño responsivo para diferentes dispositivos.

Autenticación y Hosting

Login seguro implementado con OAuth 2.0 a través de Firebase Authentication. La aplicación está hospedada en Firebase Hosting, garantizando alta disponibilidad y rendimiento.

Integración de Pagos

Implementación del sistema de pagos con PayPal para suscripciones premium, facilitando el acceso a funcionalidades adicionales y mejoras en la experiencia del usuario.

@GetMapping("/consultar")
public ResponseEntity<?> consultarMensajes() {
    List<Mensaje> mensajes = mensajeRepository.findAll();

    if (mensajes.isEmpty()) {
        return ResponseEntity.status(HttpStatus.OK).body("Sin novedades");
    } else {
        return ResponseEntity.status(HttpStatus.OK).body(mensajes);
    }
}

public void procesarMensajesCadaHora() {
    LocalTime horaActual = LocalTime.now();
    LocalTime horaLimite = horaActual.minusHours(1);
    
    logger.info("Mensaje del servidor: ","Se depuró la base de datos a las: " + horaActual);

    List<Mensaje> mensajes = mensajeRepository.findByHoraPublicacionBefore(horaLimite);
    
    // Filtrar anuncios para eliminar
    List<Anuncio> anunciosParaEliminar = listaAnuncios.stream()
            .filter(anuncio -> anuncio.getHoraEnvio().isBefore(horaLimite))
            .collect(Collectors.toList());
    
    // Eliminar anuncios y procesar mensajes
    for (Anuncio anuncio : anunciosParaEliminar) {
        
        listaAnuncios.remove(anuncio);
    }

    // Procesar mensajes
    for (Mensaje mensaje : mensajes) {
        guardarEnArchivo(mensaje);
        mensajeRepository.delete(mensaje);
    }

}

@Override
protected List<Mensaje> doInBackground(Void... voids) {
    try {
        
        URL url = new URL("XXXXXXX");
        HttpURLConnection connection = (HttpURLConnection) url.openConnection();

        // Configurar la conexión
        connection.setRequestMethod("GET");

        // Verificar la respuesta del servidor
        int responseCode = connection.getResponseCode();
        if (responseCode == HttpURLConnection.HTTP_OK) {
            // Éxito
            InputStream inputStream = connection.getInputStream();
            BufferedReader reader = new BufferedReader(new InputStreamReader(inputStream));
            // Parsear la respuesta (puedes usar Gson u otras bibliotecas)
            List<Mensaje> mensajes = parsearRespuesta(reader);
            inputStream.close();
            return mensajes;
        } else {
            // Manejar el error, si es necesario
            Log.e("HTTP", "Error al consultar mensajes: " + responseCode);
        }
        connection.disconnect();
    } catch (Exception e) {
        e.printStackTrace();
    }

    return null;
}

@Override
protected void onPostExecute(List<Mensaje> mensajes) {
    super.onPostExecute(mensajes);
    if (callback != null) {

    }
}

// Método para parsear la respuesta del servidor (puedes usar Gson u otras bibliotecas)
private List<Mensaje> parsearRespuesta(BufferedReader reader) {
    StringBuilder respuestaJson = new StringBuilder();
    String linea;
    try {
        while ((linea = reader.readLine()) != null) {
            respuestaJson.append(linea);
        }
    } catch (IOException e) {
        e.printStackTrace();
    }

    return new Gson().fromJson(respuestaJson.toString(), new TypeToken<List<Mensaje>>() {}.getType());
}
